# Discovering Context

Learn before asking. Stop when the next action is clear.

## Check

1. Read project instructions and nearby docs first.
2. Inspect relevant files, tests, examples, and existing patterns.
3. Search for names, errors, routes, commands, or concepts before guessing.
4. Note contradictions between user claims, docs, and code.
5. Ask only if missing context changes behavior, API, data, risk, or scope.

## Output

State only what matters:

- facts found;
- constraints or patterns;
- contradiction, if any;
- next action.

Do not keep exploring to feel thorough. Route to `grill-me`, `planning-work`, `debugging-work`, or direct action when enough is known.
